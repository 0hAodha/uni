Working with Git & GitHub

Introduction to Git and GitHub

  • Git:
    • A distributed version control system to track changes in your code.
    • Allows collaboration with multiple developers and maintains a history of changes.
  • GitHub:
    • A cloud-based platform for hosting Git repositories.
    • Provides collaboration tools like pull requests and issue tracking.

Step 3: Setting Up Git

  • Installing Git:
    1. Check if Git is installed:
      git --version
    1. If not installed, download it from git-scm.com for your operating system.
    1. Install Git by following the installer instructions.
  • Set Global Git Configuration:
    • Set your username and email (these will appear in your commits):
      git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

Step 4: Initialise a Local Repository

  • Create a Git Repository Locally:
    1. Open your terminal or command prompt.
    1. Navigate to the folder where you want to store your project:
      cd path/to/your/folder
    1. Initialise the Git repository:
      git init
    1. Add a README.md file and commit it:
      echo "# My Project" >> README.md
git add README.md
git commit -m "Initial commit with README"

Step 5: Creating a GitHub Repository

  • Steps to Create a New Repository on GitHub:
    1. Log in to GitHub and click the + sign in the top-right corner.
    1. Select New Repository.
    1. Name your repository, provide a description, and choose the visibility (public or private).
    1. Click Create Repository.
  • Push Local Repo to GitHub:
    1. Add the GitHub repository as a remote:
      git remote add origin https://github.com/yourusername/your-repo.git
    1. Push your local commits to GitHub:
      git push -u origin main

Step 6: Cloning a GitHub Repository

  • What is Cloning?
    • Cloning creates a local copy of a GitHub repository on your machine.
  • Steps to Clone a Repository:
    1. In your GitHub repository, click the Code button and copy the repository URL.
    1. In your terminal, run:
      git clone https://github.com/yourusername/your-repo.git
    1. Change into the repository directory:
      cd your-repo

Step 7: Basic Git Commands

  • Adding and Committing Changes:
    1. Modify or create new files in your repository.
    1. Stage the changes:
      git add .
    1. Commit the changes with a message:
      git commit -m "Added new features"
  • Pushing Changes to GitHub:
    • Push your changes to the remote repository:
      git push origin main

Step 8: Branching in Git

  • What is Branching?
    • A branch allows you to work on new features without affecting the main codebase.
  • Steps to Create a Branch:
    1. Create a new branch:
      git checkout -b feature-branch
    1. Work on your feature in this branch.
    1. Commit the changes and push the branch to GitHub:
      git push -u origin feature-branch

Step 9: Merging Branches

  • Merging a Branch to Main:
    1. Switch back to the main branch:
      git checkout main
    1. Merge the feature branch into main:
      git merge feature-branch
    1. Push the updated main branch to GitHub:
      git push origin main

Step 10: Collaboration with Pull Requests

  • What is a Pull Request?
    • A pull request lets you propose changes to someone’s repository and have them review and merge your changes.
  • Creating a Pull Request:
    1. Fork another repository on GitHub.
    1. Make changes in your forked version and push the changes.
    1. Go to the original repository and click New Pull Request.
    1. The repository owner reviews and merges your changes.
